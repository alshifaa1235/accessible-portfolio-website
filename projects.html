My Projects

Here are three projects I've been working on as part of my learning. They're not perfect — there's always something I'd do differently now — but each one taught me something new.

Project list

Personal Portfolio Website

Status: Completed

This is the portfolio site you're looking at right now. I built it from scratch using only HTML5 and CSS3 — no frameworks, no JavaScript. The main goal was to practise semantic HTML and learn how to make a website accessible.

I spent a lot of time on the navigation and making sure it works well on mobile. I also learned about ARIA labels and focus styles while building this.

  • HTML5
  • CSS3
  • Flexbox
  • Grid
  • ARIA

Student Management System

Status: Completed

A command-line application I made for a college assignment using Python. It lets you add, view, update, and delete student records stored in a text file. Nothing fancy, but it was my first time writing a proper CRUD application.

I learned a lot about file handling in Python and how to organise code into functions. I also added basic input validation so it doesn't crash if you type the wrong thing.

  • Python
  • File I/O
  • CLI
  • CRUD

Weather Information App

Status: In Progress

A small web page that shows current weather information for a city entered by the user. I'm using a free weather API and fetching data with JavaScript's fetch API. This is my first project that talks to an external service.

I'm still working on the error handling (what happens if the city name is wrong, or if there's no internet connection) and making the layout look decent on small screens.

  • HTML5
  • CSS3
  • JavaScript
  • REST API
  • Fetch API

Things I Learned Along the Way

Each project taught me something I didn't expect. The portfolio taught me how much work goes into making a simple site accessible. The student management system showed me why planning the data structure before you start coding saves a lot of time. The weather app introduced me to asynchronous programming, which was confusing at first but starting to make sense now.

I'm planning to build more projects — probably something that combines a front-end interface with a small back-end once I've learned enough Python web frameworks.
